KDP Optimizer AI — Documentación del Sistema
Descripción General
KDP Optimizer AI es un sistema de optimización de metadata para Amazon Kindle Direct Publishing (KDP) que cubre 8 mercados internacionales. Utiliza la API de DeepSeek (modelo deepseek-chat, compatible con el SDK de OpenAI) para analizar manuscritos y generar metadata específica para cada mercado.

Arquitectura del Pipeline
El proceso completo sigue este flujo:

Manuscrito (texto) 
  → [PASO 1] Análisis del manuscrito (analyzeManuscript)
  → [PASO 2] Para cada mercado:
       → Generación de metadata (generateMetadata)
       → Optimización de keywords (optimizeKeywordsForMarket)
       → Validación KDP (kdp-validator)
       → Generación de SEO (generateSEO)
  → [PASO 3] Kit de Marketing Orgánico (generateMarketingKit)
  → [PASO 4] Contenido para Landing Page (generateLandingPageContent)
  → Resultado final

Optimización de Tokens: Muestreo Estratégico del Manuscrito
Antes de enviar el texto a la IA, el sistema aplica muestreo estratégico para reducir el consumo de tokens en ~85%.

Umbral: 60,000 caracteres

Manuscritos cortos (≤60k chars): Se envía el texto completo.
Manuscritos largos (>60k chars): Se extraen 3 secciones:
Inicio: Primeros 25,000 caracteres (introducción, personajes, conflicto inicial)
Medio: 20,000 caracteres del centro del texto (desarrollo, giros de trama)
Final: Últimos 15,000 caracteres (clímax, resolución)
Total enviado: ~60,000 caracteres
Las secciones se concatenan con separadores:

=== INICIO DEL MANUSCRITO ===
[texto]
=== SECCIÓN MEDIA DEL MANUSCRITO ===
[texto]
=== FINAL DEL MANUSCRITO ===
[texto]

Manejo de Errores en Respuestas JSON
La función safeParseJSON() protege contra respuestas malformadas de la IA:

Nivel 1: Eliminar markdown fences (```json ... ```)
Nivel 2: Extraer bloque entre primer { y último }
Nivel 3: Reparar comillas tipográficas, trailing commas, claves sin comillas
Fallback: Retorna {} y registra el error en consola
Validación de Metadata KDP
Después de la generación, el sistema aplica validaciones:

Regla	Límite
Título + Subtítulo	Máximo 200 caracteres
Cada campo de keyword backend	Máximo 50 caracteres
Keywords: términos prohibidos	"bestseller", "free", "gratis", marcas registradas
Descripción HTML	Solo tags permitidos: <b>, <i>, <u>, <br>, <p>, <h4>, <h5>, <h6>, <ul>, <li>, <ol>
Rate Limiting y Delays
Para evitar errores de límite de tasa de la API:

Entre llamadas	Delay
Metadata → Keywords (mismo mercado)	600ms
Después de SEO	400ms
Entre mercados	800ms
Después de marketing kit	500ms
Después de landing page	400ms
El sistema también implementa retry automático con backoff exponencial para errores de red y rate limits.
